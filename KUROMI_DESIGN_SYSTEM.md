# 库洛米 UI 更新：设计规范与配色方案

**版本**: v2.5.0-kuromi
**文档生成日期**: 2026-01-15
**作者**: Manus AI

---

## 1. 设计理念

本次视觉更新的核心设计理念是**“甜酷” (Sweet & Cool)**。它融合了库洛米角色的两大标志性特征：

- **酷 (Cool)**: 体现在深邃的背景色、哥特风格的图标（如城堡、骷髅）以及简洁利落的布局上。
- **甜 (Sweet)**: 体现在明亮的点缀色（如粉色）、可爱的核心形象以及充满情感的细节（如心形、蝴蝶结）上。

整体风格旨在打造一个既神秘、个性，又不失可爱与少女心的视觉体验。

---

## 2. 配色方案

我们为库洛米主题定义了一套全新的配色系统。这套系统将应用于应用的亮色模式 (Light Mode) 和暗色模式 (Dark Mode)。

### 2.1. 核心调色板

| 颜色名称 | HEX 色值 | RGB | 描述 |
| :--- | :--- | :--- | :--- |
| **Kuromi Purple** | `#a155e7` | 161, 85, 231 | **主色调**。用于按钮、图标、滑块等核心交互元素。 |
| **Devil Pink** | `#e04f8f` | 224, 79, 143 | **辅助色/强调色**。用于高亮状态、通知、以及需要特别突出的元素。 |
| **Deep Black-Purple**| `#1a101f` | 26, 16, 31 | **背景色**。营造深邃、神秘的氛围。 |
| **Light Lilac** | `#f5eefc` | 245, 238, 252 | **前景色/文本色**。用于标题和正文，确保在深色背景上的可读性。 |
| **Muted Purple** | `#a28ab8` | 162, 138, 184 | **点缀色/次要文本色**。用于副标题、提示文字和禁用状态。 |
| **Pure Black** | `#000000` | 0, 0, 0 | 用于纯黑背景或阴影。 |
| **Pure White** | `#ffffff` | 255, 255, 255 | 用于高亮文本或图标。 |

### 2.2. 应用示例

- **按钮**: 背景为 `Kuromi Purple`，文本为 `Light Lilac`。
- **导航栏**: 背景为 `Deep Black-Purple`，选中项为 `Devil Pink`，未选中项为 `Muted Purple`。
- **卡片/容器**: 背景为 `rgba(26, 16, 31, 0.8)` (带透明度的深黑紫)，边框为 `Muted Purple`。
- **标题**: `Light Lilac`。
- **正文**: `Muted Purple`。

## 3. 字体规范

- **UI 字体**: 继续使用系统默认字体（San Francisco for iOS, Roboto for Android），以保证最佳的可读性和性能。
- **品牌/情感字体**: 对于特定的品牌署名（如 `by Jason Tsao...`），可以引入一款具有**霓虹灯 (Neon)** 或**哥特手写体**风格的自定义字体，以增强主题感。该字体需打包到应用资源中。

---

## 4. 图标规范

- **来源**: 所有主题图标均来自用户提供的 `kuromi-ui-kit.png`。
- **风格**: 保持图标的“描边 + 色块填充”风格。
- **尺寸**: 
  - 导航栏图标: 24x24
  - 设置项图标: 28x28
  - 按钮内嵌图标: 20x20
- **颜色**: 图标颜色应与配色方案保持一致。例如，在紫色按钮上的图标应为白色或淡紫色。

---

## 5. 布局与间距

- **栅格系统**: 遵循 8pt 栅格系统。所有组件的尺寸和间距都应为 8 的倍数（8, 16, 24, 32...），以保持视觉上的和谐与一致性。
- **圆角**: 广泛使用较大的圆角半径，以契合库洛米的可爱风格。
  - 按钮: `borderRadius: 16`
  - 卡片: `borderRadius: 24`
  - 输入框: `borderRadius: 12`

---

## 6. 动画与交互

- **核心原则**: 动画应**快速、流畅且有弹性**。
- **弹跳效果 (Spring)**: 在按钮点击、模块加载等交互中，广泛使用弹簧动画，增加趣味性。
- **淡入淡出 (Fade)**: 页面切换和内容加载时，使用平滑的淡入淡出效果。
- **触觉反馈 (Haptics)**: 在所有关键交互（如按钮点击、滑块拖动）中加入触觉反馈，提升操作的确认感和沉浸感。

---

## 7. 实现指南

建议在项目中创建一个 `styles/` 目录，并在其中定义全局的设计规范。

- **`styles/colors.ts`**: 定义全局的配色方案。
- **`styles/typography.ts`**: 定义字体大小、行高等规范。
- **`styles/spacing.ts`**: 定义间距和圆角等常量。

通过这种方式，可以确保整个应用的设计语言高度统一，并方便未来的维护和迭代。
