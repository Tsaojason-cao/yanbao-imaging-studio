# Yanbao EAS Build App - 设计文档

## 应用概述

这是一个演示 EAS Build 与 GitHub 持续集成的示例应用，集成 Supabase 后端服务。应用展示如何配置自动化构建流程、代码同步和发布管理。

## 屏幕列表

### 1. Home (首页)
- **主要内容**：应用介绍、EAS Build 状态展示
- **功能**：
  - 显示当前构建配置信息
  - 展示 Supabase 连接状态
  - 显示最近的构建历史
  - 快速访问 GitHub 仓库链接

### 2. Build Status (构建状态)
- **主要内容**：实时构建状态和历史记录
- **功能**：
  - 显示当前构建进度
  - 构建历史列表（成功/失败状态）
  - 构建日志查看
  - 下载构建产物链接

### 3. Settings (设置)
- **主要内容**：应用配置和环境信息
- **功能**：
  - 显示 Supabase 配置信息
  - 显示 Expo 项目配置
  - GitHub 仓库信息
  - 环境变量状态检查

## 主要用户流程

1. **查看构建状态流程**
   - 用户打开应用 → Home 屏幕
   - 点击 "Build Status" → 查看构建列表
   - 点击具体构建 → 查看详细日志

2. **检查配置流程**
   - 用户打开应用 → Home 屏幕
   - 点击 "Settings" → 查看配置信息
   - 验证 Supabase 和 GitHub 连接状态

## 配色方案

- **Primary (主色)**：`#0a7ea4` - 科技蓝，代表构建和自动化
- **Success (成功)**：`#22C55E` - 绿色，表示构建成功
- **Error (错误)**：`#EF4444` - 红色，表示构建失败
- **Warning (警告)**：`#F59E0B` - 橙色，表示构建进行中
- **Background (背景)**：浅色模式 `#ffffff`，深色模式 `#151718`
- **Surface (卡片)**：浅色模式 `#f5f5f5`，深色模式 `#1e2022`

## 设计原则

1. **信息清晰**：构建状态一目了然，使用颜色和图标快速识别
2. **实时反馈**：显示实时构建进度和状态更新
3. **易于导航**：底部 Tab 导航，快速切换主要功能
4. **响应式设计**：适配不同屏幕尺寸，优先考虑移动端竖屏体验

## 技术集成

- **Supabase**：后端数据存储和实时订阅
- **EAS Build**：自动化构建服务
- **GitHub Actions**：CI/CD 工作流
- **Expo Router**：文件系统路由
- **NativeWind**：Tailwind CSS 样式
